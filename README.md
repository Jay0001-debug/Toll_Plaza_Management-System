# Toll Plaza Management System

## About the Project

The **Toll Plaza Management System** is a database management project designed to simplify and organize toll plaza operations.

The system provides a centralized database for managing toll plazas, toll booths, staff, vehicles, toll rates, toll transactions, monthly passes, FASTag accounts, and vehicle violations.

## Why This Project?

Toll plazas handle a large amount of information every day, including vehicle records, toll payments, staff assignments, FASTag transactions, monthly passes, and violations.

Managing this information manually can result in data duplication, inconsistency, and difficulty in tracking records. This project provides a structured database solution to store and manage toll plaza information efficiently.

## What Does It Do?

The system can:

- Manage toll plaza and toll booth information.
- Store staff details, roles, salaries, and shift timings.
- Assign staff members to toll booths and toll plazas.
- Store vehicle and vehicle owner information.
- Define toll rates for different vehicle types.
- Record toll transactions and payment methods.
- Manage monthly toll passes and their expiry dates.
- Manage FASTag account details and balances.
- Record vehicle violations and penalty amounts.
- Track the toll rate applied to each transaction.
- Maintain organized relationships between all toll plaza operations.

## Database Design

The database is designed using relational database concepts such as:

- Primary Keys
- Foreign Keys
- Candidate Keys
- Composite Keys
- Functional Dependencies
- Database Normalization
- Boyce-Codd Normal Form (BCNF)

All relations in the database have been checked for **BCNF** to reduce data redundancy and maintain data consistency.

## Main Entities

The system includes the following main entities:

- Toll Plaza
- Toll Booth
- Staff
- Vehicle
- Toll Rate
- Transaction
- Monthly Pass
- FASTag Account
- Violation

It also includes associative relations to connect transactions with monthly passes, FASTag accounts, and toll rates.

## Purpose

The main purpose of this project is to demonstrate how a relational database can be used to manage real-world toll plaza operations.

The project focuses on efficient data organization, transaction tracking, staff management, vehicle management, toll collection, FASTag management, monthly passes, and violation tracking.

## Conclusion

The **Toll Plaza Management System** provides an organized and reliable database structure for managing toll plaza operations.

It helps maintain accurate records, reduce data redundancy, and efficiently manage toll-related information in a centralized system.
